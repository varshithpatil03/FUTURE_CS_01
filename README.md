# FUTURE_CS_01
# 🔐 DVWA Vulnerability Assessment Report

## 📖 Project Overview
This project contains the results of a vulnerability assessment performed on the Damn Vulnerable Web Application (DVWA) using OWASP ZAP.

The objective of this assessment was to identify security misconfigurations and potential vulnerabilities in a web application hosted locally.

---

## 🎯 Objective
To perform automated vulnerability scanning and analyze security weaknesses in DVWA using OWASP ZAP.

---

## 🛠 Tools Used
- OWASP ZAP v2.17.0
- DVWA (Damn Vulnerable Web Application)
- Apache Web Server
- Kali Linux / Localhost Environment

---

## 🌐 Target Information
- Target URL: http://127.0.0.1
- Application: DVWA
- Testing Type: Active & Passive Scan
- Scan Date: 28 February 2026

---

## 📊 Vulnerability Summary

| Risk Level | Count |
|------------|--------|
| High       | 0      |
| Medium     | 4      |
| Low        | 5      |
| Informational | 3   |
| **Total Alerts** | **12** |

---

## 🚨 Key Vulnerabilities Identified

### 🟠 Medium Risk
- Content Security Policy (CSP) Header Not Set
- Directory Browsing Enabled
- HTTP Only Site (No HTTPS)
- Missing Anti-clickjacking Header

### 🟡 Low Risk
- Cookie Without HttpOnly Flag
- Cookie Without SameSite Attribute
- X-Content-Type-Options Header Missing
- Server Version Information Disclosure

### 🔵 Informational
- Authentication Request Identified
- Session Management Token Identified
- User Agent Fuzzer Detection

---

## 🔍 Methodology
1. Configured DVWA locally.
2. Intercepted traffic using OWASP ZAP.
3. Performed Active and Passive scanning.
4. Analyzed alerts generated by the scanner.
5. Created structured vulnerability assessment report.
